## Title + Subtitle formula (iOS)

```
Title (30 char):    Brand name + keyword primario se possibile
Subtitle (30 char): Value proposition + keyword secondario
```
Ogni carattere è indicizzato. Non sprecare nessuno.

## Keyword strategy

- Apple: campo 100 char, no spazi tra keyword, virgole come separatori
- Includere keyword NON nel title/subtitle (già indicizzati lì)
- Target: media competizione + alta rilevanza > alto volume + bassa rilevanza
- Aggiornare ogni 4-6 settimane basandosi su dati performance

## Struttura descrizione

```
Prime 3 righe (sopra fold):  hook + value prop primario + social proof
Body:                         feature come benefici ("Risparmia 2h/settimana" non "Scheduling automatizzato")
Fine:                         CTA + link supporto
```
Mai keyword stuffing — Apple e Google penalizzano.

## Strategia screenshot

```
Screenshot 1: value prop più compelling + product UI
Screenshot 2-4: top 3 use case in contesto reale
Screenshot 5: social proof (recensioni, press, user count)
Caption: corta, benefit-focused, font grande
```

## Gestione recensioni

**Rispondere a TUTTE le recensioni negative entro 48h.**

Formula risposta:
1. Riconoscere il problema specifico
2. Empatizzare (no "mi dispiace che si senta così")
3. Soluzione o prossimo passo
4. Invito a contattare il supporto

Mai difensivi. Mai dismissivi. Ringraziare le positive con callbacks specifici.
