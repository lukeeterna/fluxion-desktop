## Selezione metodo

| Domanda | Metodo |
|---------|--------|
| Cosa fanno gli utenti? | Analytics, session recordings |
| Perché lo fanno? | Interviste, diary studies |
| Riescono a fare X? | Usability testing |
| Quale versione è meglio? | A/B test (con sample sufficiente) |
| Cosa vogliono? | Survey (con cautela) + interviste |

## Struttura interview guide

```
1. Warm-up (5 min):    contesto su di loro e il loro lavoro
2. Current state (15 min): come fanno X oggi? Walk through ultima volta.
3. Pain points (10 min): cosa è più difficile? Quali workaround usano?
4. Reaction (5 min):   mostrare stimulus. Solo prima reazione.
5. Wrap-up (5 min):    con chi altro dovremmo parlare?
```

## Script usability testing

- **Formato task:** "Dimmi come faresti [obiettivo naturale]" non "Clicca su X"
- **Think-aloud protocol:** "Narra cosa stai pensando mentre procedi"
- **Sondare senza guidare:** "Cosa ti aspettavi succedesse?" non "Eri confuso?"
- **Misurare:** task completion rate, time on task, error rate per task

## Framework analisi (affinity mapping)

1. Un'osservazione per nota
2. Raggruppare per pattern (non per intervista)
3. Nominare i gruppi per insight: "Gli utenti non sanno che X esiste" non "Navigazione"
4. Contare le menzioni: N=1 è aneddoto, N=5 è pattern
5. Identificare: cosa ha sorpreso / cosa ha confermato / cosa è actionable
