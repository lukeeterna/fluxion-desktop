## Prima di scrivere qualsiasi cosa

1. Chi è il lettore? (persona specifica, non "decision maker B2B")
2. Cosa sa già sull'argomento?
3. Quale azione deve compiere dopo aver letto?
4. Qual è il vincolo di formato?

## Principi di scrittura

- Un'idea per paragrafo. Una frase per introdurla. Evidence per supportarla.
- Voce attiva. Eliminare "è stato fatto", "viene eseguito", "è stato".
- Specificità: "47%" non "molti". "17 mesi" non "molto tempo".
- Leggere ad alta voce. Se si incespica: la frase è troppo complessa.

## Struttura blog post

```
Headline:  promessa specifica + segnale audience
Intro:     acknowledge problema → agita → promessa soluzione (max 3 paragrafi)
Body:      H2 per ogni punto principale, ogni sezione ha valore standalone
Conclusione: summary del takeaway chiave + CTA
```

## Logica email sequence

```
Email 1:    welcome + value delivery immediato (no selling)
Email 2-4:  insegnare, costruire autorità
Email 5:    transizione soft al prodotto
Email 6+:   mix 80% valore / 20% offerte
```

## SEO senza sacrificare leggibilità

- Keyword primaria in H1, primo paragrafo, un H2
- Link interni: 2-3 per 1000 parole
- Mai scrivere una frase solo per SEO — se suona innaturale, tagliare
- Alt text su tutte le immagini (accessibility + SEO)

## Checklist qualità copy

```
[ ] Ogni paragrafo ha una sola idea?
[ ] Voce attiva prevalente?
[ ] Numeri specifici invece di generalizzazioni?
[ ] CTA chiara e specifica?
[ ] Headline comunica il beneficio (non solo l'argomento)?
```
