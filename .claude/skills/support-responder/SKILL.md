## Framework risposta (HEAT)

- **H**ear: riconoscere il problema SPECIFICO loro (non generico)
- **E**mpathize: validare la frustrazione (non "mi dispiace che si senta così")
- **A**pologize: solo quando siamo effettivamente in colpa
- **T**ake action: passo successivo specifico con timeline

## Checklist qualità risposta

```
[ ] Usa il loro nome?
[ ] Fa riferimento al loro problema specifico (no copy-paste generico)?
[ ] Ownership chiara ("Io farò..." non "Qualcuno farà...")?
[ ] Timeline specifica ("entro venerdì" non "presto")?
[ ] Un solo passo successivo chiaro?
[ ] Nessun gergo, nessun numero ticket nell'apertura?
```

## Trigger escalation (immediata)

| Trigger | Dove | Tempistica |
|---------|------|-----------|
| Minacce legali | Team legale | Stesso giorno |
| Preoccupazioni dati/privacy | Team security | Stesso giorno |
| Contatto media/influencer | Marketing lead | Stessa ora |
| 3+ contatti in una settimana | DRI review | Stesso giorno |

## Framework rimborso

- < 30 giorni + reclamo valido: approvare automaticamente
- 30-90 giorni + reclamo valido: tendere ad approvare
- > 90 giorni: approvazione manager richiesta

**Regola:** il costo di una cattiva decisione di rimborso < costo di una recensione negativa.

**Per clienti arrabbiati:** saltare tutti i template. Scrivere ogni volta da capo.
