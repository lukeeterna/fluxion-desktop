### Auto-fixed Issues

**1. [Rule 1 - Bug] SPOSTAMENTO TF-IDF false positive for "devo cambiare l'olio"**

- **Found during:** Task 2 (run full pytest suite on iMac)
- **Issue:** `test_cambiare_olio_not_spostamento` failed — "devo cambiare l'olio" was classified as SPOSTAMENTO by the TF-IDF semantic layer (confidence 0.468 >= threshold 0.45). The regex pattern matcher correctly returned None (no booking object present), but the semantic layer overrode it. This is a regression from f02.1-01's SPOSTAMENTO hardening: we tightened regex patterns to require booking objects, but the TF-IDF model (trained with "cambiare" as a SPOSTAMENTO signal) was bypassing that guard.
- **Fix:** In `classify_intent()`, added a guard before returning the semantic result: if `semantic_result.category == SPOSTAMENTO` AND no regex pattern matched (or pattern result was not SPOSTAMENTO) AND `semantic_result.confidence < 0.6`, fall through to the lower-threshold pattern check instead of returning the semantic result. This preserves all existing behavior for non-SPOSTAMENTO semantics and for cases where a regex pattern did confirm the intent.
- **Files modified:** `voice-agent/src/intent_classifier.py`
- **Verification:** Re-ran full iMac suite — 1259 PASS / 0 FAIL. User manually confirmed "cambiare le gomme" routes to booking_waiting_name (NOT SPOSTAMENTO).
- **Committed in:** `e7af96b` (Task 2 commit)
