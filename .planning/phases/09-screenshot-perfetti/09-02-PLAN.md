<task type="auto">
  <name>Task 1: Automated quality audit of all 23+ screenshots</name>
  <files>landing/screenshots/*.png</files>
  <read_first>
    - landing/screenshots/ (list all PNG files to audit)
  </read_first>
  <action>
Run an automated audit on every PNG in `landing/screenshots/` (excluding `_old/` directory and `.DS_Store`).

**Check 1 — Dimensions (automated):**
For each PNG file:
```bash
sips -g pixelWidth -g pixelHeight landing/screenshots/*.png
```
Every file MUST be at least 1200 pixels wide and 800 pixels tall. Record any files that fail.

**Check 2 — File size sanity (automated):**
Any PNG smaller than 50KB is suspicious (likely corrupt or blank). Any PNG larger than 1MB is fine (detailed screenshot).
```bash
ls -la landing/screenshots/*.png
```
Flag files under 50KB.

**Check 3 — Visual inspection (read each file):**
For each of the 23+ screenshots, open/read the PNG and check:
- Does it show the FLUXION app UI (dark sidebar, content area)?
- Is the data in Italian (Italian names like Anna, Marco, Giulia — not John, Jane)?
- Are prices in euro format (e.g., "45,00" or "€45")?
- Is there any OS chrome visible (macOS menu bar at top, Dock at bottom, notification center)?
- Is there any loading spinner or skeleton placeholder visible?
- Is the content fully rendered (no blank white areas where data should be)?

**Expected files (23+):**
01-dashboard, 02-calendario, 03-clienti, 04-servizi, 05-operatori, 06-fatture, 07-cassa, 08-voice, 09-fornitori, 10-analytics, 11-impostazioni, 12-scheda-parrucchiere, 13-scheda-fitness, 14-scheda-estetica, 15-scheda-medica, 16-scheda-fisioterapia, 17-scheda-odontoiatrica, 18-scheda-veicoli, 19-scheda-carrozzeria, 20-scheda-selector, 21-trasformazioni-prima-dopo, 22-pacchetti, 23-fedelta

Create a simple audit report listing:
- PASS / FAIL for each screenshot
- Reason for any FAIL
- Total: X/23 passed

Print the report to stdout so the human can review.

  <verify>
The audit report is printed showing PASS/FAIL for each screenshot. All dimension checks run without error.
  </verify>
