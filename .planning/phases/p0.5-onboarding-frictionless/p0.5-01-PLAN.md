<objective>
Create the Voice Agent settings panel in Impostazioni so users can configure (or reconfigure) the Groq API key after initial setup.

Purpose: Users who skipped Step 8 of the wizard, or who need to rotate their key, currently have no way to configure the Groq key post-install. This is a sales blocker — if they click away from setup without entering the key, Sara never works.

Output: A new VoiceAgentSettings.tsx component and its integration in Impostazioni.tsx. No new Rust code needed — save_setup_config already saves groq_api_key to the DB, and http_bridge.rs already returns it to the voice agent.
</objective>

  <action>
Create `src/components/impostazioni/VoiceAgentSettings.tsx`.

Model it after SmtpSettings.tsx (same dark-slate Card style). Structure:

1. Header: title "Assistente Vocale Sara" with microphone icon (use Mic from lucide-react), subtitle "Configura la chiave API per abilitare l'assistente vocale 24/7".

2. Status indicator row: Show current state as a colored badge:
   - If key is loaded and starts with "gsk_": green badge "Attivo"
   - If key is empty/missing: amber badge "Non configurata"

3. Istruzioni collapsible box (always visible, compact):
   - "1. Vai su console.groq.com/keys" with external link button that opens in browser via Tauri shell `open` command: `import { open } from '@tauri-apps/plugin-shell'; open('https://console.groq.com/keys')`
   - "2. Clicca 'Create API Key', dai il nome 'Fluxion', copia la chiave"
   - "3. Incolla qui sotto (inizia con gsk_)"

4. Input field: type="password" for groq_api_key, placeholder "gsk_xxxxxxxxxxxxxxxxxxxx", show/hide toggle button.

5. Action row with two buttons:
   - "Testa" button: validates format (must start with "gsk_"), tries `fetch('http://127.0.0.1:3002/health', {signal: AbortSignal.timeout(4000)})` — on success shows "Voice agent raggiungibile", on network error shows "Formato valido — Sara partira al prossimo avvio" (normal, voice agent may not be running).
   - "Salva" button: calls `saveConfig.mutateAsync` with a SetupConfig where only groq_api_key is set (spread existing config from useSetupConfig, then override groq_api_key). Show success toast via a simple inline message "Chiave salvata" in green for 3 seconds.

6. State: use `useState` for localKey (controlled input), testStatus ('idle'|'testing'|'ok'|'error'), testMsg, saveMsg.

7. On component mount: pre-fill localKey from `useSetupConfig().data?.groq_api_key ?? ''`.

TypeScript strict: no `any`. The SetupConfig type already has groq_api_key?: string | undefined — use it directly. Use `useSetupConfig` and `useSaveSetupConfig` from hooks/use-setup.ts.

Import: use `@tauri-apps/plugin-shell` for the external link. If the import fails type-check, use `window.open('https://console.groq.com/keys', '_blank')` as fallback (note: Tauri webview may block window.open for external URLs — prefer plugin-shell if available in the project).

Check if `@tauri-apps/plugin-shell` is in package.json before using it — if not present, use a simple `<a href="..." target="_blank" rel="noopener noreferrer">` anchor styled as a button instead.
